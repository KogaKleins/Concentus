# Glossário

| Termo | Definição |
|---|---|
| Concentus | Plataforma multi-orquestra; não é o nome de nenhuma orquestra específica |
| Admin master/dev | Única conta técnica com autoridade sobre a plataforma e todas as orquestras |
| Maestro/admin | Maior nível operacional dentro de uma orquestra; os dois títulos têm as mesmas capacidades |
| Peso administrativo | Ordem de autoridade entre maestros/admins; maior administra menor, iguais coadministram |
| Membro | Participante ativo de uma orquestra |
| Líder de naipe | Membro com autoridade contextual em um ou mais naipes |
| Orquestra / tenant | Unidade isolada de dados, configuração e acesso, apresentada como contexto sem reconfigurar a identidade visual do app |
| Espaço | Contêiner de membros, bibliotecas e comunicados: global, naipe ou temporário |
| Naipe | Espaço permanente associado a instrumento ou família instrumental |
| Voz | Parte musical ordenada dentro de um naipe, como 1ª ou 2ª voz |
| Formação padrão | Estrutura atual de naipes, vozes e atribuições usada ao criar uma obra |
| Plano de distribuição | Fotografia editável da formação e dos destinatários de uma obra |
| Biblioteca | Contêiner configurável de pastas e obras |
| Obra | Registro musical identificado por número dentro de uma biblioteca e título |
| Material | Parte, áudio, documento ou outro item pertencente a uma obra |
| Anexo/arquivo | Binário físico associado a material ou comunicado |
| Concessão de acesso | Regra que liga um recurso a pessoa, voz, naipe, espaço ou público global |
| Rascunho | Recurso preparado, mas invisível ao público final |
| Publicação em lote | Operação identificada que publica vários materiais de uma obra |
| Compartilhados comigo | Área para recursos concedidos diretamente ao usuário |
| Confirmação de ciência | Ação explícita que registra que um usuário declarou estar ciente |
| Impersonação | Sessão técnica em que o admin master representa um usuário sem conhecer sua senha |
